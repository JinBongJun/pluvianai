# AgentGuard 용량 계획 (Capacity Planning)

초기 설정으로 지원 가능한 사용자 수와 리소스 사용량을 계산합니다.

## 초기 설정 리소스

### Vercel (프론트엔드) - 무료 플랜
- **대역폭**: 100GB/월
- **함수 실행 시간**: 100GB-hours/월
- **빌드 시간**: 6,000분/월

### Railway (백엔드) - $5-20/월
- **CPU**: 512MB-1GB RAM
- **PostgreSQL**: 1GB 저장 공간 (무료) 또는 5GB ($5/월)
- **Redis**: 선택사항
- **네트워크**: 무제한

---

## 사용자 행동 가정

### 일반적인 사용 패턴
- **사용자당 프로젝트**: 1-2개
- **사용자당 LLM API 호출**: 100-1,000회/월
- **사용자당 월 방문**: 10-20회
- **프로젝트당 평균 API 호출**: 50-500회/월

### 데이터 크기 가정
- **API 호출 저장 데이터**: 5-10KB/호출 (request + response JSON)
- **프론트엔드 페이지 로드**: 500KB-1MB/회
- **API 응답 크기**: 10-50KB/요청

---

## 용량 계산

### 1. 프론트엔드 대역폭 (Vercel 무료 100GB/월)

**계산:**
- 페이지 로드: 1MB/회
- 사용자당 월 방문: 15회
- 사용자당 월 대역폭: 15MB
- **최대 사용자**: 100GB ÷ 15MB = **약 6,600명**

**실제 여유를 고려하면:**
- **안전한 사용자 수**: **3,000-5,000명**

### 2. 백엔드 API 호출 (Railway)

**계산:**
- 사용자당 월 API 호출: 500회 (평균)
- API 응답 크기: 30KB/요청
- 사용자당 월 대역폭: 500 × 30KB = 15MB
- Railway 무제한 네트워크 → **네트워크 제한 없음**

**CPU/메모리 제한:**
- 동시 요청 처리: 50-100 req/s (512MB RAM 기준)
- 사용자당 평균 요청: 0.02 req/s (500회/월 ÷ 30일 ÷ 24시간 ÷ 3600초)
- **이론적 최대**: 100 ÷ 0.02 = **5,000명**

**실제 여유를 고려하면:**
- **안전한 사용자 수**: **1,000-2,000명**

### 3. 데이터베이스 저장 공간

**계산:**
- API 호출 저장: 8KB/호출 (평균)
- 사용자당 월 API 호출: 500회
- 사용자당 월 저장: 500 × 8KB = 4MB
- **1GB 저장 공간으로**: 1GB ÷ 4MB = **250명/월**

**하지만 데이터는 누적됩니다:**
- 3개월 보관: 250명 ÷ 3 = **약 80명**
- 6개월 보관: 250명 ÷ 6 = **약 40명**

**5GB 저장 공간 ($5/월)으로:**
- 3개월 보관: **약 400명**
- 6개월 보관: **약 200명**

### 4. 동시 사용자

**계산:**
- 사용자당 일일 활성 시간: 1시간
- 사용자당 월 활성 일수: 10일
- 동시 사용자 비율: 5-10%
- **최대 동시 사용자**: 1,000명 × 10% = **100명**

**Railway 512MB RAM 기준:**
- 동시 연결: 50-100개
- **안전한 동시 사용자**: **50-100명**
