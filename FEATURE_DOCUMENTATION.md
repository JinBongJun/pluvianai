# AgentGuard 기능 설명서

## 📊 프로젝트 Overview 페이지

### 1. 주요 통계 카드 (Stats Cards)

#### Total API Calls
- **설명**: 지난 7일간 프로젝트에서 발생한 모든 LLM API 호출 횟수
- **데이터 소스**: `APICall` 모델에서 `project_id`와 날짜 범위로 집계
- **표시**: 숫자 (예: 1,234)
- **업데이트**: 실시간 (API 호출 시 자동 증가)

#### Avg Quality Score
- **설명**: 지난 7일간 평가된 API 응답의 평균 품질 점수
- **데이터 소스**: `QualityScore` 모델의 `overall_score` 평균값
- **표시**: 백분율 (예: 85.5%)
- **계산 방식**: 
  - 규칙 기반 평가: JSON 유효성, 필수 필드 존재 여부, 길이 적절성, 포맷 유효성
  - 고급 평가 (Startup 이상): 의미론적 일관성, 톤 일관성, 응집성 (LLM 기반)
- **N/A 표시**: 평가된 API 호출이 없을 때

#### Total Cost
- **설명**: 지난 7일간 프로젝트의 총 LLM API 사용 비용
- **데이터 소스**: `APICall`의 토큰 수를 기반으로 제공자별 가격 계산
- **표시**: 달러 (예: $12.34)
- **계산 방식**: 
  - Provider별 가격표 기반
  - Input tokens + Output tokens × 모델별 가격
- **$0.00 표시**: API 호출이 없거나 비용 데이터가 없을 때

#### Success Rate
- **설명**: 지난 7일간 API 호출의 성공률
- **데이터 소스**: `APICall`의 `status_code` 분석
- **표시**: 백분율 (예: 98.5%)
- **계산 방식**: 
  - 성공: `status_code`가 200-299 범위
  - 실패: 그 외의 모든 경우
  - Success Rate = (성공한 호출 수 / 전체 호출 수) × 100
- **N/A 표시**: API 호출이 없을 때

### 2. 차트 (Charts)

#### Quality Scores
- **설명**: 시간에 따른 품질 점수 추이를 시각화
- **데이터**: `QualityScore` 모델의 시간별 `overall_score`
- **표시**: 
  - 데이터 있음: 라인 차트
  - 데이터 없음: "No quality data available" 메시지
- **기능**: 
  - 시간 범위 선택 (7일, 30일 등)
  - 호버 시 상세 점수 확인

#### Drift Detections
- **설명**: LLM 응답의 드리프트(변화) 감지 결과
- **데이터**: `DriftDetection` 모델
- **표시**: 
  - 데이터 있음: 드리프트 이벤트 타임라인
  - 데이터 없음: "No drift detections" 메시지
- **드리프트 유형**:
  - Basic (Free/Indie): 길이, 포맷 변화
  - Enhanced (Startup 이상): 의미론적, 톤 변화

### 3. 탭 (Tabs)

#### Overview (현재 탭)
- 프로젝트 통계 및 차트 표시
- 모든 멤버가 조회 가능

#### Team Members
- 프로젝트 멤버 목록 및 관리
- 역할별 권한 관리

#### Settings
- 프로젝트 설정 (이름, 설명 수정)
- Owner/Admin만 접근 가능

## 👥 Team Members 기능

### 멤버 역할 (Roles)

#### Owner (소유자)
- **권한**:
  - ✅ 프로젝트 삭제
  - ✅ 프로젝트 설정 변경
  - ✅ 멤버 추가/제거/역할 변경
  - ✅ 모든 데이터 조회 및 수정
- **특징**: 
  - 프로젝트 생성 시 자동 할당
  - 역할 변경 불가
  - 제거 불가

#### Admin (관리자)
- **권한**:
  - ✅ 프로젝트 설정 변경
  - ✅ 멤버 추가/제거/역할 변경
  - ✅ 모든 데이터 조회 및 수정
- **제한**:
  - ❌ 프로젝트 삭제 불가
  - ❌ Owner 역할 변경 불가

#### Member (멤버)
- **권한**:
  - ✅ 프로젝트 데이터 조회
  - ✅ API 호출 생성 (프록시 사용)
  - ✅ 품질 평가 실행
- **제한**:
  - ❌ 프로젝트 설정 변경 불가
  - ❌ 멤버 관리 불가

#### Viewer (뷰어)
- **권한**:
  - ✅ 프로젝트 데이터 조회만 가능
- **제한**:
  - ❌ 모든 수정 작업 불가
  - ❌ API 호출 생성 불가

### 멤버 관리 기능

#### 멤버 추가
1. "Add Member" 버튼 클릭
2. 이메일 주소 입력
3. 역할 선택 (Admin/Member/Viewer)
4. 추가 완료

**주의사항**:
- 추가하려는 사용자가 AgentGuard에 가입되어 있어야 함
- 이미 멤버인 사용자는 추가 불가
- Owner는 추가 불가 (이미 Owner)

#### 역할 변경
- Owner/Admin만 가능
- 드롭다운에서 역할 선택
- 즉시 적용

#### 멤버 제거
- Owner/Admin만 가능
- 확인 후 제거
- 제거된 멤버는 프로젝트 접근 불가

## 📈 데이터 흐름

### API 호출 모니터링
1. **프록시 요청**: 클라이언트 → AgentGuard 백엔드 → LLM Provider
2. **로깅**: 요청/응답이 `APICall` 테이블에 저장
3. **비용 계산**: 토큰 수 기반으로 자동 계산
4. **품질 평가**: 선택적으로 `QualityScore` 생성
5. **드리프트 감지**: 배치 작업으로 주기적 분석

### 통계 집계
- **실시간**: API 호출 시 즉시 카운트 증가
- **배치**: 품질 점수, 드리프트는 주기적 계산
- **캐싱**: 자주 조회되는 통계는 Redis에 캐시 (5분 TTL)

## 🎯 사용 시나리오

### 시나리오 1: 새 프로젝트 시작
1. Dashboard에서 "Create Project" 클릭
2. 프로젝트 이름과 설명 입력
3. Overview 페이지에서 통계 확인 (초기에는 모두 0)
4. API 프록시를 통해 LLM 호출 시작
5. 통계가 자동으로 업데이트됨

### 시나리오 2: 팀 멤버 초대
1. 프로젝트 → Team Members 탭
2. "Add Member" 클릭
3. 팀원 이메일 입력 및 역할 선택
4. 팀원이 로그인하면 프로젝트 접근 가능

### 시나리오 3: 품질 모니터링
1. Overview에서 Avg Quality Score 확인
2. Quality Scores 차트에서 추이 확인
3. 점수가 낮으면 API 호출 상세 확인
4. 필요시 품질 평가 재실행

### 시나리오 4: 비용 최적화
1. Total Cost 카드에서 비용 확인
2. Cost Analysis API로 모델별 비용 비교
3. 비용이 높은 모델 식별
4. 더 저렴한 모델로 전환 검토

## ⚠️ 주의사항

1. **데이터 보관 기간**: 구독 플랜에 따라 다름 (Free: 7일, Indie: 30일 등)
2. **API 호출 제한**: 월별 제한이 있으며, 초과 시 업그레이드 필요
3. **멤버 제한**: 프로젝트당 멤버 수는 플랜에 따라 제한됨
4. **품질 평가**: 고급 평가는 Startup 이상 플랜 필요
5. **드리프트 감지**: Enhanced 드리프트는 Startup 이상 플랜 필요

## 🔧 문제 해결

### 통계가 0으로 표시되는 경우
- API 호출이 실제로 발생했는지 확인
- 날짜 범위 확인 (기본값: 지난 7일)
- 프로젝트 ID가 올바른지 확인

### 멤버 추가 실패
- 사용자가 AgentGuard에 가입되어 있는지 확인
- 이미 멤버인지 확인
- 이메일 주소가 정확한지 확인

### 품질 점수가 N/A인 경우
- API 호출에 대한 품질 평가가 실행되었는지 확인
- 품질 평가 API를 수동으로 호출해보기
