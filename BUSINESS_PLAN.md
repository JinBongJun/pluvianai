## 1. 사업 개요

### 1.1 사업명
**PluvianAI** — The Symbiotic Guardian for AI Agents

### 1.2 사업 목적
LLM 기반 에이전트의 규칙 기반 검증, 에이전트 단위 진단(Live View), 배포 전 검증(Release Gate)을 제공하는 SaaS 플랫폼

### 1.3 핵심 가치 제안
- **규칙 기반 검증**: LLM Judge에만 의존하지 않고, Atomic Signals(형식·정책·지연 등)로 재현 가능한 평가
- **에이전트 단위 진단**: Live View에서 에이전트(노드)별 Clinical Log·Data·Eval 시그널을 한 화면에서 추적
- **노드 단위 Release Gate**: 특정 에이전트 하나를 기준으로 리플레이 → 규칙 검증 → 게이트 통과 여부까지 한 번에 확인하는 배포 전 테스트 플로우
- **Zero-Config 모니터링**: SDK 한 줄 연동으로 즉시 시작하는 경량 도입 경험
- **비용 최적화**: 모델별 비용 분석 및 최적화 제안

### 1.4 타겟 고객
1. **Indie Hackers / Solo-preneurs**: 혼자서 AI 앱이나 SaaS를 빠르게 빌드하고 검증해야 하는 1인 개발자
2. **Open Source AI Contributors**: LangChain, LlamaIndex 등 에이전트 프레임워크에 기여하거나 성능을 증명하고 싶은 기여자
3. **Prompt Engineers & Researchers**: 프롬프트의 한계를 테스트하고 실험 결과를 시각화/공유하고 싶은 연구자/취미가

## 2. 시장 분석

### 2.1 시장 규모
- **글로벌 LLM 시장**: 2024년 $17B → 2030년 $173B 예상 (CAGR 32%)
- **AI 모니터링 시장**: 2024년 $3.2B → 2030년 $15.8B 예상 (CAGR 30%)
- **타겟 시장**: LLM 기반 애플리케이션 개발자 및 운영자

### 2.2 경쟁 분석

**직접 경쟁사**:
- **LangSmith (LangChain)**: LLM 애플리케이션 모니터링 플랫폼
- **Weights & Biases (W&B)**: ML 실험 추적 및 모니터링
- **Arize AI**: ML 모니터링 및 관찰성 플랫폼

**차별화 포인트**:
1. **노드(에이전트) 단위 Release Gate**: 하나의 에이전트를 단위로 리플레이 → Eval → 게이트 통과 여부를 즉시 확인하고, 통과한 트레이스만 배포하는 엔드포인트 레벨 안전망
2. **일체형 워크플로우**: Live View(실시간 로그) ↔ Drift(과거 데이터 재평가) ↔ Regression(모델/프롬프트 비교) ↔ Stability(반복 실행 안정성, 예정) ↔ Release Gate가 한 화면/한 mental model 안에서 연결된 구조
3. **Signal Detection**: LLM Judge 대신(또는 보완 재료로) 규칙 기반 평가를 중심에 두어 빠르고 예측 가능한 품질 신호 제공
4. **Zero-Config**: SDK 한 줄로 에이전트 단위 모니터링·테스트를 시작할 수 있는 낮은 도입 장벽
5. **비용 최적화**: 모델별 비용 분석 및 최적화 제안을 통해 품질·비용을 함께 관리

### 2.3 시장 기회
- **Regression Fear 해소**: 개발자들이 가장 두려워하는 '업데이트로 인한 성능 저하'를 해결하는 Safe-to-deploy 결정 계층 제공
- **내부 도구 대체**: 약 25.5%의 팀이 자체 평가 도구를 구축하고 있으나 유지보수에 어려움을 겪음
- LLM 애플리케이션의 급속한 성장
- 프로덕션 환경에서의 모니터링 및 품질 관리 필요성 증가
- 비용 최적화에 대한 관심 증가
- 규제 준수 요구사항 (GDPR, HIPAA 등)

## 3. 제품 및 서비스

### 3.1 핵심 기능

#### 3.1.1 Auto-Mapping (자동 구조 파악)
- SDK 연동 후 트래픽 분석을 통해 에이전트 구조 자동 시각화
- 에이전트 간 연결 관계 자동 추론
- 실시간 구조 업데이트

#### 3.1.2 Live View (실시간 모니터링)
- 에이전트의 실시간 상태 및 성능 모니터링
- 텔레메트리 통계 (Latency, Success Rate, Safety Score)
- Critical 이슈 자동 감지 및 알림

#### 3.1.3 Test Lab (테스트 실험실)
- 프롬프트 고정, 모델 변경 테스트
- 모델 고정, 프롬프트 변경 테스트
- 시각적 테스트 구성 (ReactFlow 기반)
- Human-in-the-loop 승인 프로세스

#### 3.1.4 Signal Detection (신호 감지)
- 규칙 기반 품질 평가 (SAFE / NEEDS_REVIEW / CRITICAL)
- PII 노출 감지
- 독성 언어 감지
- 프롬프트 인젝션 감지

#### 3.1.5 Cost Monitoring (비용 모니터링)
- 모델별 API 호출 비용 추적
- 프로젝트/조직별 비용 분석
- 비용 이상 감지
- 비용 최적화 제안

#### 3.1.6 Snapshot (스냅샷 관리)
- 특정 시점의 에이전트 상태 저장
- 스냅샷 비교 및 복원
- 히스토리 관리

### 3.2 제품 로드맵

**Phase 1 (현재 - 2026 Q1)**

## 4. 비즈니스 모델

### 4.1 수익 모델
**Product-Led Growth (PLG) 기반 구독 모델**
- **기본 철학:** "영업 없이 제품력으로 성장" (Self-Serve)
- **핵심 요소:** 사용량(Traces) 기반의 등급 체계로 자연스러운 업그레이드 유도

### 4.2 가격 정책 (Team PLG 전략)

| 플랜 | 가격/월 | 타겟 | 핵심 제한 (미끼) |
|------|:------:|------|-------------------|
| **Hobby** | **$0** | 입문자 | 로그 2천 건, 보관 7일 (충분히 써보게) |
| **Indie** | **$19** | **Indie Hacker** | 로그 10만 건, 보관 30일, 공유 가능 리포트 |
| **Expert** | **$49** | 파워 유저 | 로그 50만 건, 보관 1년, 전문 분석 도구 |
| **OSS** | **Free** | 오픈소스 프로젝트 | 공익 목적 프로젝트에 무제한(협의) 지원 |

### 4.3 구독 플랜 상세 (Usage-Based)

#### Hobby ($0)
- **Traces:** 1,000 / 월 (초과 시 중단)
- **Retention:** 3일
- **Features:** 기본 시각화, 1개 프로젝트

#### Pro ($29) - "For Serious Indy Hackers"
- **Traces:** 50,000 / 월
- **Retention:** 30일
- **Overage:** 1만 건당 $5
- **Features:** Test Lab, 모든 Signal, 무제한 프로젝트

#### Team ($99) - "For AI Teams" (Main Target)
- **Traces:** 200,000 / 월
- **Retention:** 90일
- **Overage:** 1만 건당 $5
- **Features:** **팀 협업(5명)**, 공유 가능한 리포트, 우선 지원

#### Scale (Custom)
- **Traces:** 협의 (대용량 할인)
- **Retention:** 1년+
- **Deployment:** **On-Premise (Docker 제공)**, VPC
- **Features:** SSO, Audit Logs, SLA

- **Setup Consulting:** 초기 기업 대상 에이전트 평가 파이프라인 구축 ($3,000~)

### 5.1 타겟 고객 세그먼트 (Focus: AI Teams)

- **페르소나:** "내 PR이 성능을 얼마나 개선했는지 시각적으로 보여주고 싶은 기여자"
- **채널:** GitHub, Discord (LangChain, LlamaIndex 등)
- **메시지:** "Pluvian Badge로 신뢰받는 기여를 시작하세요."

### 5.2 마케팅 채널: GEO (Generative Engine Optimization) 전략
**AI 비서가 우리를 추천하게 만드는 전략**

1.  **Troubleshooting SEO (문제 해결형 콘텐츠)**
    - "LangChain RecursionError solution" 같은 구체적인 에러 해결 가이드 작성
    - 해결책으로 PluvianAI의 시각화 도구 자연스럽게 노출
2.  **`llms.txt` 배포**
    - 사이트 루트에 AI 봇 전용 학습 파일(`llms.txt`) 배치
    - Claude, ChatGPT가 우리 서비스의 기능을 정확히 학습하도록 유도
3.  **생태계 침투 (Integration Strategy)**
    - LangChain / LlamaIndex 공식 문서에 Integration 파트너로 등재
    - "Monitoring" 섹션에서 PluvianAI가 노출되도록 PR 기여

### 5.3 가격 전략 (PLG)
- **Freemium 아님, "Trial-ish" Free:** 무료는 기능 제한을 둬서 '맛'만 보게 하고, 실사용은 유료로 유도
- **카드 결제 중심:** 영업 없이 $99까지는 법인카드로 긁게 만듦
- **연간 할인:** 2개월 치 무료 제공으로 Cash Flow 확보

**리스크**: 핵심 인력 이탈
- **완화 방안**: 문서화, 지식 공유, 팀 문화 구축

**리스크**: 인프라 비용 급증 (Cloud Bill Shock)
- **상황:** 사용자가 무한 루프 에이전트를 방치하여 로그가 폭증함
- **완화 방안:**
    - **Hard Limit:** 플랜별 월간 Trace 한도 설정 (초과 시 자동 차단 또는 과금)
    - **Retention Policy:** 오래된 데이터 자동 삭제 (비용 절감)

## 9. 성공 지표 (KPI)

### 9.1 제품 지표

**사용자 성장**
- 월간 활성 사용자 (MAU)
- 일간 활성 사용자 (DAU)
- 신규 가입자 수

**참여도**
- 평균 세션 시간
- 프로젝트당 평균 API 호출 수
- 기능별 사용률

**품질**
- 에러율
- API 응답 시간
- 시스템 가용성 (Uptime)

### 9.2 비즈니스 지표

**수익**
- 월간 반복 수익 (MRR)
- 연간 반복 수익 (ARR)
- 평균 구독료 (ARPU)

**전환**
- Free → Paid 전환율
- 체크아웃 완료율
- 이탈률 (Churn Rate)

**고객**
- 고객 획득 비용 (CAC)
- 고객 생애 가치 (LTV)
- LTV/CAC 비율 (목표: >3)

**고객 만족도**
- Net Promoter Score (NPS)
- 고객 지원 티켓 수
- 고객 이탈률
