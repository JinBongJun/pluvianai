# AgentGuard 인프라 비용 분석

## 📊 현재 인프라 구성

### 현재 사용 중인 서비스

- **Frontend**: Vercel (Next.js)
- **Database**: PostgreSQL (Railway 제공)

## ⚠️ 비용 위험 요소

### 1. Railway의 숨겨진 비용

**문제점**:
- 무료 티어는 매우 제한적
- 실제 서비스 시작 시 즉시 Pro 플랜 필요
- 스토리지/네트워크 초과 시 예상치 못한 비용 발생

**해결책**:
- 데이터 아카이빙 (오래된 데이터는 압축/삭제)
- R2 같은 저렴한 스토리지로 오래된 데이터 이동
- 데이터 보관 정책 설정 (예: 90일 이후 아카이브)

### 2. Supabase vs Railway 비교

#### Supabase 무료 티어
- **500MB 데이터베이스**
- **1GB 파일 스토리지**
- **2GB 대역폭**
- **제한**: 프로젝트 2개

#### Supabase Pro 플랜 ($25/월)
- **8GB 데이터베이스**
- **100GB 파일 스토리지**
- **250GB 대역폭**
- **추가**: $0.125/GB (DB), $0.021/GB (스토리지)

#### 비교 분석

| 항목 | Railway Pro | Supabase Pro |
|------|------------|--------------|
| 기본 비용 | $20/월 | $25/월 |
| DB 스토리지 | 10GB 포함 | 8GB 포함 |
| 파일 스토리지 | 별도 필요 | 100GB 포함 |
| Redis | 포함 | 별도 필요 |
| 확장성 | 좋음 | 좋음 |

**결론**: Railway가 약간 저렴하지만, 파일 스토리지가 필요하면 Supabase가 유리

## 💡 비용 최적화 전략

### 1. Cloudflare R2 통합 (권장)

#### R2 장점
- **무료 티어**: 10GB 스토리지, 1M Class A 작업/월
- **저렴한 가격**: $0.015/GB/월 (S3 대비 80% 저렴)
- **대역폭 무료**: egress 비용 없음
- **S3 호환**: 기존 코드 재사용 가능

#### 구현 전략
1. **Hot Data**: PostgreSQL (최근 30일)
2. **Warm Data**: R2 (30-90일, 압축)
3. **Cold Data**: R2 (90일+, 아카이브)

#### 예상 비용 절감
- **현재**: 50GB DB = $10/월 (Railway)
- **R2 사용**: 10GB DB + 40GB R2 = $0.60/월
- **절감**: **$9.40/월 (94% 절감)**

### 2. 데이터 아카이빙 정책

```python
# 예시: 90일 이후 데이터는 R2로 이동
- 0-30일: PostgreSQL (빠른 쿼리)
- 30-90일: R2 (압축, 필요시 다운로드)
- 90일+: R2 아카이브 (거의 접근 안 함)
```

### 3. Redis 최적화

- **TTL 설정**: 자주 사용하지 않는 캐시는 짧은 TTL
- **메모리 제한**: 최대 메모리 설정으로 OOM 방지
- **선택적 캐싱**: 중요한 쿼리만 캐싱
