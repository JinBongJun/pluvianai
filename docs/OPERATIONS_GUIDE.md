# 운영 가이드 (Operations Guide)

## 개요

AgentGuard 서비스의 일일/주간/월간 운영 체크리스트와 문제 해결 가이드입니다.

## 일일 체크리스트 (5분)

### 1. 모니터링 대시보드 확인
- [ ] Vercel 대시보드: 프론트엔드 상태 확인
- [ ] Railway 대시보드: 백엔드 및 데이터베이스 상태 확인
- [ ] Sentry: 에러 발생 여부 확인
- [ ] Health Check 대시보드 (`/admin/health`): 시스템 상태 확인

### 2. 알림 확인
- [ ] 이메일 알림: Critical 알림 확인
- [ ] Slack/Discord: 중요한 알림 확인

### 3. 빠른 상태 확인
- [ ] 메인 대시보드 접속 확인
- [ ] API 응답 시간 확인
- [ ] 주요 기능 동작 확인

## 주간 체크리스트 (15분)

### 1. 성능 메트릭 리뷰
- [ ] API 호출 통계 확인
- [ ] 비용 분석 리뷰
- [ ] 에러율 트렌드 확인
- [ ] Drift 감지 결과 확인

### 2. 리소스 사용량
- [ ] 데이터베이스 크기 확인
- [ ] 메모리 사용량 확인
- [ ] 디스크 사용량 확인

### 3. 알림 설정 검토
- [ ] 알림 채널 설정 확인
- [ ] 알림 임계값 검토
- [ ] 불필요한 알림 필터링

## 월간 체크리스트 (30분)

### 1. 비용 분석
- [ ] 월간 총 비용 확인
- [ ] 예산 대비 비용 검토
- [ ] 비용 최적화 기회 검토
- [ ] 인프라 비용 확인 (Railway/Vercel)

### 2. 백업 확인
- [ ] 데이터베이스 백업 확인
- [ ] 백업 복구 테스트 (선택적)

### 3. 보안 검토
- [ ] API 키 로테이션 검토
- [ ] 사용자 접근 권한 검토
- [ ] 보안 업데이트 확인

## 데이터베이스 백업

### Railway 백업 확인

1. Railway 대시보드 접속
2. 데이터베이스 서비스 선택
3. "Backups" 탭 확인
4. 최근 백업 날짜 확인

### 백업 복구 프로세스

1. Railway 대시보드에서 백업 선택
2. "Restore" 옵션 선택
3. 복구 시점 선택
4. 확인 후 복구 실행

**주의**: 백업 복구는 데이터 손실을 초래할 수 있으므로 신중하게 진행해야 합니다.

## 문제 해결 가이드

### 문제 1: 데이터베이스 연결 실패

**증상**: Health Check에서 데이터베이스 상태가 "error"

**해결 방법**:
1. Railway 대시보드에서 데이터베이스 상태 확인
2. 데이터베이스 연결 정보 확인
3. 환경 변수 `DATABASE_URL` 확인
4. 필요 시 데이터베이스 재시작

### 문제 2: 높은 CPU/메모리 사용량

**증상**: Health Check에서 CPU 또는 메모리 사용량이 90% 이상

**해결 방법**:
1. Railway 대시보드에서 리소스 사용량 확인
2. 최근 배포 확인 (최근 변경사항이 원인일 수 있음)
3. API 호출 패턴 확인 (비정상적인 트래픽 확인)
4. 필요 시 스케일링 고려

### 문제 3: 비용 급증

**증상**: 월간 비용이 예상보다 많이 증가

**해결 방법**:
1. Cost Analysis 페이지에서 비용 트렌드 확인
2. 모델별 비용 확인 (특정 모델 사용량 급증 확인)
3. Shadow Routing 최적화 검토
4. Cost Optimizer 제안 확인

### 문제 4: Drift 감지

**증상**: Drift Detection에서 품질 하락 감지

**해결 방법**:
1. Drift 상세 페이지에서 변화 확인
2. 모델 성능 변화 확인
3. 데이터 분포 변화 확인
4. 필요 시 모델 업데이트 또는 조정

## 자동화된 모니터링

서비스는 다음 자동화된 모니터링을 제공합니다:

1. **Hourly Health Check**: 매시간 시스템 상태 확인
2. **Daily Drift Detection**: 매일 2 AM UTC에 Drift 감지
3. **Daily Cost Anomaly Detection**: 매일 3 AM UTC에 비용 이상 감지
4. **Daily Infrastructure Cost Check**: 매일 9 AM UTC에 인프라 비용 확인

Critical 알림은 자동으로 이메일로 발송됩니다.

## 롤백 프로세스

### 코드 롤백 (Vercel/Railway)

1. 배포 히스토리에서 이전 버전 선택
2. "Rollback" 또는 "Redeploy" 클릭
3. 롤백 완료 후 확인

### Shadow Routing 롤백

1. Shadow Routing 설정 페이지로 이동
2. "Rollback" 버튼 클릭
3. 확인 후 롤백 실행
