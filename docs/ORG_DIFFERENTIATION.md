# AgentGuard Org 페이지 차별화 전략

## 목표
- 표준 SaaS 패턴을 따르되, AgentGuard만의 LLM 모니터링 특화 정보를 강조
- 다른 서비스(Vercel, Supabase)와 차별화된 사용자 경험 제공

---

## 1. 다른 서비스들과의 비교

### Vercel (배포 플랫폼)
- **중점**: 배포, 도메인, 함수 호출
- **정보**: 배포 상태, 함수 호출 수, 대역폭 사용량
- **특징**: 배포 중심 대시보드

### Supabase (Backend-as-a-Service)
- **중점**: 데이터베이스, 스토리지, API 호출
- **정보**: DB 크기, API 요청 수, 스토리지 사용량
- **특징**: 인프라 중심 대시보드

### AgentGuard (LLM 모니터링)
- **중점**: LLM API 호출, 품질, 드리프트, 비용
- **정보**: API Calls, Quality Score, Drift 감지, Cost 분석
- **특징**: 모니터링 및 분석 중심 대시보드

---

## 2. AgentGuard만의 특화 정보

### 2-1. Organizations 리스트 화면 (`/organizations`)

**표준 정보** (다른 서비스와 동일):
- Org 이름
- 플랜 정보
- 프로젝트 개수

**AgentGuard 특화 정보** (추가):
- 📊 **총 API Calls**: Org 내 모든 프로젝트의 최근 7일 API 호출 수
- 💰 **총 비용**: Org 내 모든 프로젝트의 최근 7일 비용 합계
- ⚠️ **활성 Alerts**: Org 전체의 미해결 알림 수
- 🔄 **Drift 감지**: 드리프트가 감지된 프로젝트 수

**Org 카드 예시**:
```
┌────────────────────────────────────────────────────────────────────┐
│ [아이콘]  My Company Org                    [+ New organization] │
│          Pro Plan • 5 projects                                    │
│          📊 45,234 calls  💰 $128.50  ⚠️ 3 alerts  🔄 1 drift    │
└────────────────────────────────────────────────────────────────────┘
```

### 2-2. Projects 화면 (`/organizations/{orgId}/projects`)

**표준 정보** (다른 서비스와 동일):
- 프로젝트 리스트
- 프로젝트 이름, 설명
- 생성일, 업데이트일

- 🔄 **Drift**: 드리프트 감지 여부 (있으면 표시)

## 3. 차별화 포인트 요약

### 3-1. 표준 패턴 유지 (사용자 친화성)
- ✅ Org 리스트 화면 구조
- ✅ Org 생성 플로우
- ✅ Settings 구조 (Team, Billing 등)
- ✅ 사이드바 네비게이션

### 3-2. AgentGuard 특화 정보 추가 (차별화)
- ✅ **LLM 모니터링 중심**: API Calls, Cost, Quality, Drift
- ✅ **실시간 상태 표시**: Alerts, Drift 감지
- ✅ **비용 분석**: 프로젝트별 비용 분해
- ✅ **품질 추적**: Quality Score 추이

### 3-3. 사용자 경험 개선
- ✅ **3초 테스트**: Org 상태를 빠르게 파악
- ✅ **문제 발견**: Alerts 섹션으로 문제 프로젝트 빠르게 확인
- ✅ **비용 관리**: Usage 섹션으로 비용 한도 파악
- ✅ **품질 모니터링**: Quality Score로 전반적인 품질 파악

## 4. 구현 우선순위

### Phase 1: 표준 구조 (기본)
1. ✅ Org 리스트 화면 (표준 정보)
2. ✅ Org 생성 폼
3. ✅ Projects 화면 기본 구조

### Phase 2: AgentGuard 특화 정보 추가
1. ✅ Org 카드에 API Calls, Cost, Alerts 표시
2. ✅ Usage 섹션 (API Calls, Cost, Quality)
3. ✅ Alerts 섹션 (드리프트, 품질, 비용 알림)
4. ✅ 프로젝트 카드에 특화 정보 표시

### Phase 3: 고급 기능
1. ✅ 실시간 업데이트 (WebSocket 또는 Polling)
2. ✅ 차트 및 트렌드 분석
3. ✅ 필터 및 정렬 (비용, 품질, 알림 기준)
