# AgentGuard 대시보드 레이아웃 와이어프레임 설계

## 목표
- **전역 → 프로젝트 → 세부** 플로우가 자연스럽게 연결되도록 설계
- 업계 표준 SaaS 대시보드 패턴 적용
- 핵심 정보를 3초 안에 파악 가능하도록 구성

---

## 1. `/dashboard` (프로젝트 목록 화면) - 실제 서비스 구조 반영

**참고한 실제 서비스들**:
- Vercel: 좌측 Usage/Alerts 사이드바 + 우측 Projects 그리드
- Supabase: 상단 검색/필터 + 프로젝트 카드 리스트
- Railway: 상단 Trial 배너 + 프로젝트 카드 (Architecture/List 뷰)

### 1-1. 실제 서비스 구조 분석

**Vercel 스타일 (2-컬럼 레이아웃)**:
- 좌측: Usage 섹션, Alerts 섹션 (고정 사이드바)
- 우측: Projects 그리드 (메인 콘텐츠)

**Supabase/Railway 스타일 (단일 컬럼)**:
- 상단: 헤더 + 검색/필터 + 뷰 전환
- 메인: 프로젝트 카드 그리드

2. **전역 요약 바를 제거** → 대신 **좌측 Usage/Alerts 섹션**으로 이동 (Vercel 스타일)

4. **상단에 검색 + 필터 + 뷰 전환** 옵션 추가 (Supabase/Railway 스타일)

### 1-3. 상단 헤더 바 (모든 서비스 공통 패턴)

**구성 요소**:
```
┌─────────────────────────────────────────────────────────────────────┐
│  Projects              [🔍 Search] [🔔 3] [👤 User Name ▼] [+ New] │
└─────────────────────────────────────────────────────────────────────┘
```

**요소별 설명**:
- **좌측**: 페이지 제목 ("Projects")
- **우측 (왼쪽부터)**:
  1. **검색 아이콘** (`🔍`): 클릭 시 전역 검색 모달 열기 (Cmd/Ctrl+K)
  2. **알림 아이콘** (`🔔 3`): 읽지 않은 알림 개수 배지 표시, 클릭 시 알림 센터 열기
  3. **유저 프로필** (`👤 User Name ▼`): 
     - 아바타 이미지 또는 이니셜
     - 사용자 이름 또는 이메일
     - 드롭다운 화살표
     - 클릭 시 드롭다운 메뉴:
       ```
       ┌─────────────────────┐
       │  👤 User Name       │
       │  user@example.com    │
       │  ─────────────────  │
       │  📊 Usage           │
       │  ⚙️  Settings       │
       │  💳 Billing         │
       │  ─────────────────  │
       │  🚪 Logout          │
       └─────────────────────┘
       ```
  4. **Create Project 버튼** (`+ New Project`): 새 프로젝트 생성

**백엔드 연결**:
- 알림 개수: `GET /notifications/unread-count`
- 유저 정보: `GET /auth/me` (이미 로드됨)
- 유저 프로필 드롭다운 메뉴:
  - Usage → `/settings/usage` (또는 Usage 대시보드)
  - Settings → `/settings`
  - Billing → `/settings/billing`
  - Logout → 로컬 스토리지 토큰 삭제 + `/login`으로 이동

### 1-4. 사이드바 하단 유저 정보 (보조 역할)

**구성 요소**:
```
┌─────────────────────┐
│  [User Info]        │
│  👤 User Name       │
│  user@example.com   │
│  [Free] 배지        │
│                     │
│  Settings           │
│  Logout             │
└─────────────────────┘
```

**역할**:
- **상단 헤더의 유저 프로필이 메인**: 빠른 접근, 드롭다운 메뉴로 모든 기능 접근
- **사이드바 하단은 보조**: 
  - 사이드바를 내릴 때 항상 보이는 유저 정보
  - 모바일/작은 화면에서 유용
  - Settings/Logout 빠른 접근 (하지만 상단 헤더에서도 가능)

**설계 원칙**:
- 상단 헤더: **주요 인터랙션** (검색, 알림, 프로필 드롭다운)
- 사이드바 하단: **보조 정보** (유저 이름, 이메일, 플랜 배지)

### 1-5. 상단 검색/필터 영역 (Supabase/Railway 스타일)

**구성 요소**:
```
┌─────────────────────────────────────────────────────────────┐
│  Projects                    [🔍 Search projects...] [Filter]│
│                                                              │
│  5 Projects    Sort: Recent Activity ▼    [Grid] [List]     │
└─────────────────────────────────────────────────────────────┘
```

**기능**:
- **검색창**: 프로젝트 이름/설명으로 실시간 필터링 (프론트엔드)
- **필터 버튼**: 고급 필터 패널 열기 (상태, 플랜, 날짜 등)
- **정렬**: "Recent Activity", "Name A-Z", "Most Calls", "Most Alerts" 등
- **뷰 전환**: Grid 뷰 / List 뷰 (Railway 스타일)

**백엔드 연결**:
- 검색: 프론트엔드 필터링 (빠른 응답) 또는 `GET /projects?search=...`
- 정렬: 프론트엔드 정렬 또는 백엔드 `GET /projects?sort_by=...&order=...`

### 1-3. 프로젝트 카드 설계 (실제 서비스 스타일 반영)

**Vercel/Supabase/Railway 스타일 - 간결한 카드**:

```
┌─────────────────────────────────────┐
│  [아이콘]  My Project    [owner]   │
│                                     │
│  Short description of the project   │
│  goes here...                       │
│                                     │
│  📊 1,234 calls  ⚠️ 2  🕐 2h ago   │
│                                     │
│                              [→]    │
└─────────────────────────────────────┘
```

**또는 더 간결한 버전 (Railway 스타일)**:

```
┌─────────────────────────────────────┐
│  [아이콘]  My Project               │
│                                     │
│  Description here...                │
│                                     │
│  📊 1,234  ⚠️ 2  🕐 2h ago         │
│                                     │
│                              [→]    │
└─────────────────────────────────────┘
```

**상태 표시 규칙 (간결하게)**:
- **하단 한 줄에 모든 상태 정보**:
  - `📊 1,234` (24h 호출 수)
  - `⚠️ 2` (알림 수, 색상으로 위험도 표시)
  - `🕐 2h ago` (마지막 업데이트)
- **색상 규칙**:
  - 알림 0개: `✅` 또는 표시 안 함
  - 알림 1-2개: `⚠️` (주황색)
  - 알림 3개 이상: `🔴` (빨간색)
- **우측 화살표**: 클릭 가능 표시 (Railway/Supabase 스타일)

**백엔드 연결**:
- 프로젝트 목록 API에 통계 포함: `GET /projects?include_stats=true`
- 응답 예시:
```json
{
  "id": 1,
  "name": "My Project",
  "description": "...",
  "stats": {
    "calls_24h": 1234,
    "alerts_count": 2,
    "last_updated": "2024-01-15T10:00:00Z"
  }
}
```

**성능 최적화**:
- 프로젝트가 많을 때: 백엔드에서 한 번에 통계 포함하여 반환
- 캐싱: 5분 TTL로 설정
- 로딩 상태: 통계가 없으면 스켈레톤 UI 표시

### 2-1. Key Metrics 카드 상세 설계

**각 카드 구조**:
```
┌─────────────────────────┐
│  API Calls              │
│                         │
│  12,340                 │
│  ↑ 12% (vs last week)   │
│                         │
│  Total calls in last 7d │
│                         │
│  [View Details →]      │
└─────────────────────────┘
```

**인터랙션**:
- **API Calls 카드**: 클릭 → `/dashboard/{projectId}/api-calls`
- **Quality 카드**: 클릭 → `/dashboard/{projectId}/quality` (또는 Quality 상세 페이지)
- **Cost 카드**: 클릭 → `/dashboard/{projectId}/cost` (또는 Cost 상세 페이지)
- **Success Rate 카드**: 클릭 → `/dashboard/{projectId}/api-calls?status=error` (에러 필터링)

**트렌드 표시**:
- 전주 대비 변화율 표시 (예: `↑ 12%`, `↓ 5.3%`)
- 색상: 증가는 녹색/파란색, 감소는 빨간색/주황색 (지표에 따라 다름)
  - API Calls 증가: 좋음 (녹색)
  - Quality 증가: 좋음 (녹색)
  - Cost 증가: 나쁨 (빨간색)
  - Success Rate 증가: 좋음 (녹색)

## 4. 구현 우선순위

### Phase 1: 핵심 기능 (즉시 구현 가능)
1. ✅ **상단 헤더 바 추가** (모든 서비스 공통 패턴)
   - 좌측: 페이지 제목
   - 우측: 검색 아이콘, 알림 아이콘(배지), 유저 프로필(드롭다운), Primary Action 버튼
   - 유저 프로필 드롭다운: Usage, Settings, Billing, Logout

2. ✅ **좌측 Usage/Alerts 섹션 추가** (Vercel 스타일)
   - Usage: API Calls, Cost, Quality (Last 7 days)
   - Alerts: Open Alerts 리스트 (최근 3-5개)
   - 각 지표 클릭 시 관련 페이지로 이동

3. ✅ **프로젝트 카드 상태 인디케이터 추가**
   - Calls (24h)
   - Alerts 수
   - Updated 시간

### Phase 2: 최적화 (성능 개선)
1. **백엔드 엔드포인트 추가**
   - `GET /api-calls/stats/global?days=7` (전역 API 호출 통계)
   - `GET /alerts/global?is_resolved=false` (전역 알림 통계)
   - `GET /projects?include_stats=true` (프로젝트 목록 + 통계 한 번에)

2. **프로젝트 카드 통계 캐싱**
   - React Query 또는 SWR 사용하여 자동 캐싱
   - 5분 TTL로 설정

### Phase 3: 고급 기능 (향후)
1. **전역 요약 바 클릭 인터랙션**
   - API Calls 클릭 → 가장 활성 프로젝트로 이동
   - Alerts 클릭 → 가장 많은 알림 프로젝트로 이동

2. **프로젝트 카드 정렬/필터**
   - "Most Active", "Most Alerts", "Recently Updated" 정렬 옵션
   - 상태별 필터 (Active, Inactive, Has Alerts 등)

## 5. 실제 서비스 구조 비교 및 개선 포인트

### 5-1. 실제 서비스 구조 분석 요약

| 서비스 | 레이아웃 특징 | 우리가 배울 점 |
|--------|--------------|---------------|
| **Vercel** | 상단 헤더(검색/알림/프로필) + 좌측 Usage/Alerts 사이드바 + 우측 Projects 그리드 | • **상단 헤더 오른쪽에 검색/알림/프로필** (모든 페이지 공통)<br>• Usage 섹션으로 전역 상태 파악<br>• Alerts 섹션으로 문제 프로젝트 빠르게 확인<br>• 프로젝트 카드 간결 (이름, URL, 최근 커밋만) |
| **Supabase** | 상단 헤더(검색/프로필) + 검색/필터 + 프로젝트 카드 리스트 | • **상단 헤더 오른쪽에 검색/프로필**<br>• 검색창이 눈에 잘 띄는 위치<br>• 프로젝트 카드에 Provider/Region 정보<br>• 간결한 상태 표시 |
| **Railway** | 상단 헤더(Help/알림/프로필) + Trial 배너 + 프로젝트 카드 (Architecture/List 뷰) | • **상단 헤더 오른쪽에 Help/알림/프로필**<br>• 뷰 전환 옵션 (Grid/List)<br>• 프로젝트 카드에 서비스 상태 표시 (2/2 online)<br>• 정렬 옵션 제공 |

### 5-2. 우리 AgentGuard에 적용할 구조 (최종)

**핵심 원칙**:
1. **상단 헤더 바** (모든 서비스 공통 패턴)
   - 좌측: 페이지 제목
   - 우측: 검색, 알림, 유저 프로필, Primary Action 버튼
   - 모든 페이지에서 일관된 위치
2. **좌측 Usage/Alerts 섹션** (Vercel 스타일)
   - 전역 상태를 한눈에 파악
   - 문제가 있는 프로젝트 빠르게 확인
3. **상단 검색/필터/정렬** (Supabase/Railway 스타일)
   - 프로젝트가 많아져도 쉽게 찾기
   - 뷰 전환 옵션 제공
4. **간결한 프로젝트 카드** (모든 서비스 공통)
   - 핵심 정보만 표시 (이름, 설명, 상태 인디케이터)
   - 클릭 가능한 화살표로 명확한 인터랙션
5. **사이드바 하단 유저 정보** (보조 역할)
   - 상단 헤더가 메인, 사이드바는 보조 정보
   - 모바일/작은 화면에서 유용

### 5-3. 현재 구조의 강점
- ✅ 사이드바 네비게이션이 명확함
- ✅ 프로젝트 카드 그리드가 직관적
- ✅ Overview의 Key Metrics 카드가 잘 정리됨

### 5-4. 추가로 개선할 점 (실제 서비스 구조 반영)
- ✅ **상단 헤더 바 추가**: 모든 페이지에서 일관된 위치에 검색/알림/프로필 배치
- ✅ **좌측 Usage/Alerts 섹션 추가**: 전역 상태를 사이드바처럼 고정 영역에 표시
- ✅ **상단 검색/필터/정렬**: 프로젝트가 많아질 때를 대비한 유틸리티
- ✅ **프로젝트 카드 간소화**: 상태 인디케이터는 유지하되, 카드 내부를 더 깔끔하게
- ✅ **뷰 전환 옵션**: Grid/List 뷰 전환 (선택사항)

### 5-5. 최종 목표 (수정)
**"3초 테스트"**: 사용자가 `/dashboard`에 들어왔을 때, 3초 안에:
1. 전체 서비스 사용량/상태 (좌측 Usage 섹션)
2. 문제가 있는 프로젝트 (좌측 Alerts 섹션)
3. 원하는 프로젝트 찾기 (상단 검색/필터)
4. 프로젝트 상태 비교 (프로젝트 카드 상태 인디케이터)

를 파악할 수 있어야 함.
