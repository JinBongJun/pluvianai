# Tech Spec: 헬스체크 엔드포인트

## 목표
- 헬스체크 엔드포인트를 제공해 서비스 상태를 빠르게 확인한다.
- DB 연결 상태를 포함해 운영 안정성을 판단할 수 있게 한다.
- 로드밸런서/모니터링이 사용할 수 있는 응답을 제공한다.

## 비목표
- 상세 성능 지표(메트릭 대시보드)
- 외부 모니터링 시스템 연동

## 요구사항
1) 공개 헬스체크 엔드포인트 제공 (`/api/v1/health`)
2) DB 연결 확인 포함 (SELECT 1)
3) 앱 이름/버전 반환
4) DB 장애 시 503 응답

## 성공 기준
- DB 정상 시 200 + `database: ok`
- DB 장애 시 503 + 명확한 오류 메시지

## 리스크/가정
- DB 연결이 느리면 헬스체크도 지연될 수 있음
