대시보드 영역 순서 (1차 집중 구간)
프로젝트 목록 대시보드 (/dashboard)
전체 구조, 사이드바, 상단 헤더
프로젝트 카드, 검색창, “Create Project” 버튼, 정렬/필터 등
프로젝트 상세 Overview (/dashboard/[projectId])
프로젝트 헤더(이름·설명·메타 정보)
탭들(Overview, API Calls, Compare, Reports, Team, Settings)
통계 카드 4개 + 각 카드의 “View Details”
Quality/Drift 차트와 인터랙션
API Calls 리스트 (/dashboard/[projectId]/api-calls)
필터 패널(열기 버튼, DateRangePicker, Provider/Model/Status/Agent/검색)
테이블(헤더 정렬, 페이지네이션)
각 row 액션(클릭/“View” 버튼), Export 버튼
API Call 상세 (/dashboard/[projectId]/api-calls/[callId])
상단 요약(Provider, Model, Status 등)
탭(Request/Response/Metadata/Quality)
이전/다음 이동, 관련 링크(품질·드리프트로 점프)
Compare 페이지 (/dashboard/[projectId]/compare)
기간 선택, 모델 선택/토글
비교 표/차트, 추천 영역, 제한(플랜) 관련 버튼
Reports 페이지 (/dashboard/[projectId]/reports)
템플릿 선택(Standard/Detailed/Executive)
Date range, “Generate Report”, “Download” 버튼
Team Members / Project Settings 탭
멤버 관리(추가/삭제/역할 변경 버튼)
프로젝트 설정(이름/설명 저장, 삭제 버튼 – Danger 영역)
공통 요소 정리
Notification 벨, Global Search, 프로필/플랜 드롭다운
키보드 단축키(Cmd/Ctrl+K, Cmd/Ctrl+N 등)
공통 모달(프로젝트 생성, 삭제 확인 등)
각 단계에서 “구조 → 기능 → 버튼/컨트롤 설명 → 백엔드 매핑 → UX 의견” 순서로 같이 볼 겁니다.
