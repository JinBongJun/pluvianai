# 바이브 코딩 잘하는 10가지 공식

> AI로 코딩하는 시대에서 실력 차이는 코드를 얼마나 잘 쓰느냐가 아니라, 에이전트를 어떻게 다루느냐에서 납니다.  
> 바이브 코딩은 '감각'이 아니라 '운영'에 가깝다는 것.  
> 키보드를 누르는 속도보다 더 중요한 건 에이전트가 헛돌지 않도록 목표·맥락·검증을 설계하는 능력입니다.

Cursor 팀이 공개한 "Best practices for coding with agents"를 기반으로 바로 적용해볼 수 있는 바이브 코딩 10가지 습관으로 정리했습니다.

---

## 핵심 철학

- **바이브 코딩은 '감각'이 아니라 '운영'과 '시스템'**
- 실력 차이는 코드 작성 능력이 아니라 **에이전트 운영 능력**에서 발생
- 키보드 속도보다 중요한 것: **목표·맥락·검증 설계 능력**

---

## 10가지 습관

### 1. 바로 치지 말고, 먼저 계획부터 세워라

잘 되는 바이브 코딩의 출발점은 **'계획 모드'**입니다.  
에이전트에게 코드를 쓰게 하기 전에, 무엇을 만들지 명확히 정의하세요.

**실행 방법:**
- 작업 시작 전 목표와 요구사항을 명확히 정리
- 구현 전략과 단계를 먼저 논의

---

### 2. Plan Mode는 옵션이 아니라 기본값이다

Plan Mode를 켜면 에이전트는 **파일 조사 → 질문 → 구현 계획**을 먼저 만듭니다.  
이 과정이 결과물의 퀄리티를 거의 결정합니다.

**실행 방법:**
- Cursor의 Plan Mode를 기본으로 활성화
- 에이전트가 계획을 세우는 과정을 검토하고 승인

---

### 3. 마음에 안 들면 고치지 말고, 다시 시작해라

중간 결과를 프롬프트로 땜질하지 마세요.  
계획을 수정하고 처음부터 다시 돌리는 게 더 빠르고 깔끔합니다.

**실행 방법:**
- 부분 수정보다는 계획 재검토 후 재시작
- 잘못된 방향으로 가면 과감히 롤백

---

### 4. 컨텍스트는 '많이'가 아니라 '정확히'다

파일을 잔뜩 던지면 오히려 성능이 떨어집니다.  
정확한 힌트만 주고, 나머지는 에이전트가 찾게 두세요.

**실행 방법:**
- 관련 파일만 선별적으로 제공
- 핵심 정보와 요구사항만 명확히 전달
- 불필요한 파일은 제외

---

### 5. 작업이 바뀌면 대화도 새로 파라

대화가 길어질수록 에이전트는 산만해집니다.  
기능 단위가 바뀌면 과감히 새 대화를 여세요.

**실행 방법:**
- 기능별로 새 대화 시작
- 관련 없는 이전 대화 내용은 참조만 활용
- 대화 히스토리 관리

---

### 6. 과거 작업은 복붙 말고 참조하라

이전 대화 전체를 붙여넣지 말고,  
Cursor의 Past Chats 같은 참조 기능을 활용하세요.

**실행 방법:**
- 이전 대화 전체 복사 대신 참조 기능 사용
- 필요한 맥락만 선택적으로 전달
- Cursor의 Past Chats 기능 활용

---

### 7. 반복되는 실수는 'Rules'로 막아라

같은 실수를 두 번 봤다면 그건 사람 문제가 아니라 규칙 문제입니다.  
룰은 짧고, 필수만 넣는 게 핵심입니다.

**실행 방법:**
- 반복되는 실수 패턴 파악
- Cursor Rules에 명확한 가이드라인 추가
- 간결하고 필수적인 규칙만 작성

**예시:**
```
- 모든 API 엔드포인트는 에러 핸들링 포함
- TypeScript 타입은 명시적으로 정의
- 테스트 코드는 실제 사용 케이스 기반으로 작성
```

---

### 8. 반복 업무는 'Skills'로 자동화하라

테스트, PR, 디펜던시 업데이트처럼 자주 하는 일은  
명령어 하나로 끝내는 게 진짜 바이브 코딩입니다.

**실행 방법:**
- 자주 반복하는 작업 패턴 식별
- Cursor Skills로 자동화 가능한 작업 정의
- 재사용 가능한 명령어 생성

**예시:**
- 테스트 작성 자동화
- PR 템플릿 생성
- 의존성 업데이트 및 검증

---

### 9. 목표가 명확하면 에이전트는 끝까지 간다

테스트, 린트, 성공 조건처럼  
"끝났는지 아닌지"가 분명해야 에이전트가 잘 일합니다.

**실행 방법:**
- 명확한 성공 기준 정의
- 테스트 케이스로 검증 가능하게
- 린트 및 코드 품질 기준 명시

**체크리스트:**
- [ ] 테스트 코드 작성 여부
- [ ] 린트 통과 여부
- [ ] 성공 시나리오 정의
- [ ] 에러 케이스 처리

---

### 10. 에이전트는 도구가 아니라 동료처럼 써라

계획을 요구하고, 설명을 시키고, 마음에 안 들면 반박하세요.  
잘 쓰는 사람일수록 에이전트를 '관리'합니다.

**실행 방법:**
- 에이전트에게 설명 요청
- 제안에 대해 질문하고 반박
- 코드 리뷰처럼 피드백 제공
- 에이전트의 의도를 이해하고 수정 요청

---

## 바이브 코딩을 '시스템'으로 만들기

### 성공하는 사람들의 공통점

AI가 똑똑해서가 아니라, **사람이 AI가 똑똑해질 수밖에 없는 환경을 만들어주기** 때문입니다.

### 5가지 핵심 원칙

1. **계획을 먼저 세우게 하기**
   - Plan Mode 활용
   - 목표와 요구사항 명확화

2. **컨텍스트를 정확히 주기**
   - 필요한 정보만 선별
   - 핵심 맥락 전달

3. **대화 노이즈를 관리하기**
   - 기능별 새 대화 시작
   - 관련 없는 내용 제거

4. **반복 실수를 규칙으로 막기**
   - Rules에 명확한 가이드라인
   - 간결하고 필수적인 규칙만

5. **성공 기준을 테스트로 고정하기**
   - 명확한 검증 방법
   - 자동화된 테스트 활용

---

## 결론

이걸 해내면 바이브 코딩은 **재현 가능한 실력**이 됩니다.

바이브 코딩을 잘하는 사람은:
- ✅ 계획을 먼저 세우게 하고
- ✅ 컨텍스트를 정확히 주고
- ✅ 대화 노이즈를 관리하고
- ✅ 반복 실수를 규칙으로 막고
- ✅ 성공 기준을 테스트로 고정한다

이러한 **시스템화된 접근**이 바로 바이브 코딩의 핵심입니다.
